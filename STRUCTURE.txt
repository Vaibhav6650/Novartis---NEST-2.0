DQI PIPELINE - COMPLETE FILE STRUCTURE
=====================================

dqi_pipeline/
│
├── __init__.py                    # Package initialization
├── __main__.py                    # Makes package executable with python -m
├── run_dqi.py                     # Main entry point script
│
├── config/
│   ├── __init__.py
│   └── dqi_config.py             # Configuration constants (weights, penalties, thresholds)
│
├── io/
│   ├── __init__.py
│   ├── data_loader.py            # Loads Excel files from study folders
│   └── result_writer.py          # Writes results to CSV files
│
├── metrics/
│   ├── __init__.py
│   ├── completeness.py           # A1-A4: Missing values, visits, domains, structural
│   ├── consistency.py            # B1-B3: Cross-domain, temporal, referential integrity
│   ├── accuracy.py               # C1-C3: Range, categorical, format compliance
│   ├── timeliness.py             # D1-D2: Entry lag, visit adherence
│   ├── integrity.py              # E1-E2: Duplicates, unique ID validity
│   ├── statistical.py            # F1-F2: Z-score outliers, IQR anomalies
│   └── ml_metrics.py             # G1: Isolation forest anomaly detection
│
├── aggregation/
│   ├── __init__.py
│   ├── subject_dqi.py            # Orchestrates all metric calculations per subject
│   ├── site_dqi.py               # Aggregates subject DQI to site level
│   └── study_dqi.py              # Aggregates site DQI to study level
│
└── pipeline/
    ├── __init__.py
    └── dqi_pipeline.py           # Main pipeline: processes all studies end-to-end

SUPPORTING FILES:
- README.md                        # Complete documentation
- requirements.txt                 # Python dependencies
